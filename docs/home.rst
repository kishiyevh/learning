UAV Mechanics: Theory & Implementation
--------------------------

This documentation presents a **technical guide** exploring
core topics in robotics, state estimation, perception, and autonomous systems.
Each topic focuses on a specific concept, combining **theoretical notes,
mathematical formulations, and practical implementations** in Python and/or C++.

We establish our foundation by replicating and running the complete navigation
pipelines of several recent, state-of-the-art AI-native autonomous UAV systems.


The goal of this technical documentation is to build a structured understanding of the
fundamental building blocks used in modern robotic systems and autonomous
vehicles, particularly those involving:

- robot kinematics and odometry
- control systems
- probabilistic state estimation
- visual perception
- sensor fusion
- navigation pipelines

Each section in the sidebar corresponds to a specific topic
and contains:

- Conceptual notes
- Mathematical explanations
- Algorithm implementations
- Experimental code snippets
- References

Project Structure
-----------------

The documentation is organized as a sequence of daily explorations:

- **— Kinematics, odometry, and basic control**  
- **— PX4 interaction and Kalman filtering fundamentals**  
- **— Camera models and feature extraction pipelines**  
- **— Feature matching, geometry estimation, and triangulation**  
- **— Coordinate transforms, state estimation, and navigation tools**  
- **— Visual odometry and sensor fusion pipelines**

Each section includes code implementations such as:

- PID simulations
- Extended Kalman filters
- feature detection and matching pipelines
- particle filters
- visual odometry front-ends
- VO-IMU fusion experiments

Technologies Used
-----------------

The implementations throughout the project rely on several commonly used
robotics and computer vision tools:

- Python
- C++
- OpenCV
- numerical optimization
- probabilistic estimation techniques
- robotics middleware concepts (TF2, navigation pipelines)

The project intentionally mixes **mathematical reasoning and code** to connect
theory with real implementation practices.
